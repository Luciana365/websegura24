Cross-Site Request Forgery (CSRF)

Diferencias y consecuencias en la seguridad

Cross-Site Request Forgery (CSRF), también conocido como ataque de solicitud falsificada de sitios cruzados, es un tipo de vulnerabilidad en la que un atacante engaña a un usuario autenticado para que realice acciones no deseadas en un sitio web vulnerable en el que el usuario tiene una sesión activa.

En un ataque CSRF, el atacante aprovecha la confianza establecida entre el usuario y un sitio web para enviar solicitudes HTTP maliciosas desde el navegador del usuario sin que éste sea consciente de ello. Estas solicitudes pueden incluir acciones como cambiar la contraseña de la cuenta del usuario, realizar compras no autorizadas u otras acciones que podrían causar perjuicios significativos.

El gran peligro del CSRF es que, al estar basado en la confianza que el usuario tiene en un sitio web específico, puede resultar difícil de detectar. Los perjuicios de un ataque CSRF pueden variar desde el robo de información confidencial hasta la modificación de datos importantes en aplicaciones web.

Ejemplo de Ataque CSRF

Supongamos que hay un usuario que tiene una cuenta en un sitio web de compras en línea y ha iniciado sesión en su cuenta, dejando la sesión activa en su navegador. El atacante crea una página web maliciosa donde incluye un formulario que realiza una acción específica en el sitio web de compras en línea que dejó abierto el usuario, por ejemplo, cambiar la dirección de envío o hacer una compra fraudulenta.

El atacante envía un enlace a la página maliciosa al usuario, aparentando que es un enlace legítimo a un sitio confiable o de interés para él, como un artículo interesante, una promoción de descuento, etc. Esto se conoce como ingeniería social.

El usuario, confiando en el aparente enlace legítimo, hace clic en él y accede a la página maliciosa creada por el atacante. La página incluye un formulario que enviará una solicitud al sitio web de compras en nombre del usuario, ya que su sesión sigue activa en el navegador.

Sin que el usuario se dé cuenta, al enviar el formulario desde la página maliciosa, se realiza una solicitud HTTP al sitio web de compras en línea con la acción deseada por el atacante, como cambiar la contraseña o realizar una compra no autorizada.

La solicitud se ejecuta con éxito en el lado del servidor del sitio web de compras en línea, ya que proviene de un usuario autenticado y con una sesión activa, por lo que el ataque de CSRF se ha llevado a cabo con éxito.

Medidas para Prevenir CSRF

  • Token CSRF: Utilizar tokens CSRF únicos y aleatorios en cada solicitud realizada por el usuario para verificar la legitimidad de la solicitud.
  • Encabezados de solicitud: Configurar encabezados de solicitud como SameSite y Content-Type para prevenir solicitudes CSRF desde otros sitios web.
  • Políticas de seguridad de origen cruzado (CORS): Implementar políticas adecuadas de CORS para limitar las solicitudes desde orígenes externos.
  • Protección basada en referencias: Verificar la presencia de encabezados de referencia válidos para asegurar que las solicitudes provengan de páginas web legítimas.

Medidas de Seguridad para Usuarios

  • Cerrar sesión: Cerrar sesión en sitios web después de utilizarlos, especialmente en dispositivos compartidos o públicos.
  • No hacer clic en enlaces sospechosos: Ser cauteloso al hacer clic en enlaces de correos electrónicos, mensajes de texto o redes sociales, especialmente de remitentes desconocidos.
  • Utilizar contraseñas seguras: Usar contraseñas fuertes y únicas para cada cuenta en línea.
  • Utilizar una solución de gestión de contraseñas: Emplear herramientas de gestión de contraseñas para mantener las credenciales seguras.
  • Autenticación de doble factor: Implementar la autenticación de doble factor para proteger las cuentas.
  • Mantener el software actualizado: Mantener el navegador y otros programas actualizados con las últimas actualizaciones de seguridad.
  • Usar extensiones de seguridad: Utilizar extensiones de seguridad en navegadores web para proteger contra ataques CSRF y otros tipos de amenazas.
